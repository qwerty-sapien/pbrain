# Evidence Base

ProductiveBrain is built on a narrow learning premise: vague intent should become one executable block, study should require retrieval, practice should be spaced and deliberate, and progress should be monitored without turning review into a gate.

The product maps those ideas to CLI behavior:

- `pb goal` and `pb plan` reduce ambiguity before execution.
- `pb study`, `pb teach`, and scoped `pb anki` favor active recall over passive rereading.
- `pb practise` turns skills into constrained reps with success checks.
- `pb finish` captures evidence while the session is still fresh.
- `pb review` summarizes and reflects without blocking the next block.

Research anchors:

- **Retrieval practice:** Classroom review literature reports consistent benefits from retrieval practice across real educational settings ([Agarwal et al., 2021](https://link.springer.com/article/10.1007/s10648-021-09595-9)).
- **Distributed practice:** A recent meta-analytic review found distributed practice outperformed massed study for curriculum-relevant classroom learning ([Mawson & Kang, 2025](https://www.mdpi.com/2076-328X/15/6/771)).
- **Self-monitoring:** A 2023 meta-analysis reported that monitoring tools can improve academic achievement ([de Bruin et al., 2023](https://link.springer.com/article/10.1007/s10648-023-09718-4)).
- **Self-regulated learning:** A 2024 systematic review links self-regulated learning to executive functions such as planning and self-monitoring ([Eckerlein et al., 2024](https://link.springer.com/article/10.1007/s10648-024-09932-8)).

The implementation should stay modest: reduce ambiguity, run a useful block, record evidence, generate scoped recall, and continue.
